  EXPLICAÇÕES TÉCNICAS (20 frases)
────────────────────────────────────────────────────────────

   46. Isso funciona assim: o modelo processa os tokens um por um e gera a resposta.
   47. A rede neural aprende padrões nos dados de treino e depois generaliza pra situações novas.
   48. O tokenizador divide o texto em pedaços menores que o modelo consegue entender.
   49. Cada camada do transformer refina a representação do texto.
   50. A atenção multi-cabeça permite que o modelo foque em diferentes partes do contexto ao mesmo tempo.
   51. O treinamento ajusta os pesos da rede pra minimizar o erro nas previsões.
   52. Quanto mais dados de qualidade, melhor o modelo aprende.
   53. A temperatura controla a criatividade das respostas. Mais alta, mais criativo.
   54. O contexto é a janela de texto que o modelo consegue ver de uma vez.
   55. Embeddings são representações numéricas das palavras num espaço vetorial.
   56. A loss é uma medida de quanto o modelo tá errando. Quanto menor, melhor.
   57. Gradient descent é o algoritmo que ajusta os pesos na direção certa.
   58. Overfitting acontece quando o modelo decora os dados em vez de aprender padrões gerais.
   59. A validação serve pra checar se o modelo tá generalizando bem.
   60. Batch size é quantos exemplos o modelo vê de uma vez durante o treino.
   61. Learning rate é a velocidade com que o modelo aprende. Muito alto ele oscila, muito baixo ele demora.
   62. A normalização ajuda a estabilizar o treino evitando que os valores fiquem muito grandes ou pequenos.
   63. Dropout é uma técnica que desliga neurônios aleatórios pra evitar overfitting.
   64. O otimizador AdamW combina momentum com correção de viés pra convergir mais rápido.
   65. Perplexidade é uma métrica que indica quão surpreso o modelo fica com os dados. Menor é melhor.
